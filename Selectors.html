7.2.2 选择用于测试的目标

一些测试指令包含有selector，用来限定可以运行测试的目标，或者声明在特定目标上测试是期望失败。

选择器为：

  • 一个或者多个目标机器三元组，可能包括通配符
  • 单个有效目标关键字（参见Effective-Target Keywords）
  • 一个逻辑表达式

根据上下文，选择器可以指定测试是否被跳过并被报告为不支持，或者期望失败。使用‘*-*-*’来匹配任意目标机器。

选择器表达式出现在大括号中，并且使用单个逻辑操作符：‘!’，‘&&’，或者‘||’。操作数可以为另一个选择器表达式，有效目标关键字，单个目标机器三元组，或者使用引号或者大括号包裹的目标机器三元组列表。例如： 

     { target { ! "hppa*-*-* ia64*-*-*" } }
     { target { powerpc*-*-* && lp64 } }
     { xfail { lp64 || vect_no_align } }
