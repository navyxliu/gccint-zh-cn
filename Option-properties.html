8.2 选项属性

     Condition({defined (USE_CYGWIN_LIBSTDCXX_WRAPPERS)})

选项记录的第二个域可以指定下列属性：

Common
选项对所有语言和目标都有效。
Target
选项对所有语言都有效，但是目标特定的。
Driver

language
当编译给定语言时，选项有效。

可以为多个不同的语言指定同一个选项。每个语言language必须已经被之前的Language记录声明过。参见Option file format。

RejectDriver
该选项仅被 （cc1.） 之类编译器属性使用，不应该被driver接受。
RejectNegative
选项没有“no-”形式。所有由“f”，“W”或者“m”开始的选项都被假设具有“no-”形式，除非使用这个属性。
Negative(othername)
选项将会关掉另一个选项othername，这是去掉前导符“-”的选项名字。这会通过Negative属性来传播一连串的选项关闭行为。
Joined
Separate
选项接受一个强制参数。Joined指示选项和参数可以被包含在同一个argv项中（例如 -mflush-func=name）。Separate指示选项和参数可以为分开的argv项（如-o）。一个选项允许同时具有这两个属性。
JoinedOrMissing

选项接受一个可选参数。如果参数给出，则会作为选项本身的argv项的一部分。

该属性不能和Joined或Separate一起使用。

MissingArgError(message)
如果一个选项被标记为 Joined 或 Separate，在强制性参数丢失的时候 message 将被当作错误信息，没有 MissingArgError 的选项将会使用通用错误。message 应该包含一个单独的 ‘%qs’ 格式，用来格式化传入的参数。
Args(n)
如果一个选项被标记为 Separate，通过 n 来区分他，默认是1。
UInteger
选项的参数是一个非负整数。选项解析器将会在传给选项处理前检测并转换参数。
NoDriverArg
如果一个选项标记为 Separate，该选项仅对编译器有效，而对driver无效。主要是为 -Wp, 之类的参数兼容性考虑，新选项没有这个属性了。
Var(var)
该选项的状态将被存储在变量var(actually a macro for global_options.x_var)中。存储状态的方式取决于选项的类型：
  • 如果使用了Mask或者InverseMask属性，则var为包含 mask的整数变量。
  • 如果选项是一个正常的on/off开关，则var为整数变量，并且当启用该选项时其值为非零。当使用选项的正面形式时，选项解析器会把变量置为1，当使用“no-”形式的时候，置为0。
  • 如果选项接受一个参数，并且具有UInteger属性，则var为整数变量，并且存储了参数的值。
  • 如果该选项接受一个参数，并具有 Enum属性， var 是一个用来存储参数值的变量（‘Enum’记录中的 Name 属性和 Enum 属性拥有同样参数的 Type 属性中的给定类型）。
  • 如果选项中有Defer属性，var 是一个指向用来存储后续过程使用的选项 VEC(cl_deferred_option,heap) 指针。（var被声明成 void * 类型，并且在使用之前需要转化成 VEC(cl_deferred_option,heap)）
  • 否则，如果选项接受一个参数，则var为指向参数字符串的指针。如果参数是可选的并且没有给出，则该指针将为null。

选项处理脚本通常把 var 初始化为0。 你可以通过修改 Init 来到达修改默认行为的目的。

Var(var, set)
选项控制一个整数变量var，并且当var等于set时起作用。当使用选项的正面形式时，选项解析器会把变量置为set，当使用“no-”形式的时候，置为!set。

var的声明方式与上面描述的单一参数形式的具有相同的方式。

Init(value)
由属性Var指定的变量应该被静态初始化为value。
Mask(name)
选项与target_flags变量（参见Run-time Target）中的一个位相关联，并且当该位被置位时才起作用。你还可以指定Var去选择一个变量，而不只是target_flags。

选项处理脚本将会自动为选项分配一个唯一的位。如果选项与‘target_flags’关联，则脚本会将宏MASK_name设为合适的位掩码。它还会声明一个TARGET_name宏，当选项起作用时其值为1，否则为0。如果使用Var将选项关联到不同的变量上，则相关的宏分别叫做OPTION_MASK_name和OPTION_name。

可以使用 MaskExists 来禁止自动位分配。

InverseMask(othername)
InverseMask(othername, thisname)
选项是具有Mask(othername)属性的另一个选项的反面。如果给出了thisname，则选项处理脚本会声明一个TARGET_thisname宏，当选项起作用时其为1，否则为0。
MaskExists
由Mask属性指定的掩码已经存在。这样，就不会有MASK和TARGET定义被增加到options.h中。

该属性的主要目的用来支持同义选项。第一个选项应该使用‘Mask(name)’并且其它的应该使用‘Mask(name) MaskExists’。

Enum(name)

该选项的参数是一个‘Enum’ 记录中对应的字符串。在纪录传递选项处理之前，该字符串被选中并根据 ‘EnumValue’中的值转化为相应整数。

Defer
这个选项应该存储为一个向量，和 Var 一样，在后续过程中使用。
Alias(opt)
Alias(opt, arg)
Alias(opt, posarg, negarg)
别名的选项 -opt。第一种形式，被传递给别名的任何参数被认为传递给了-opt，-opt 被认为是否定形式如果别名是否定形式。第二种形式，别名可能不是否定或者没有任何参数，并且 posarg 被认为当作一个参数传递给 -opt。第三中形式，别名可能没有任何参数，如果别名被在积极形式中使用，则 posarg 被视为传递给-opt，并且如果别名被在否定形式中使用，则 negarg 被视为传递给-opt。

别名不应该被制定为Var 或 Mask 或 UInteger。别名通常应该被制定为相同语言，然而这些别名在命令行是不被区分的。

当 Alias 定义被一个选项使用的时候，driver specs不必做处理，并且没有对应 ‘OPT_’ 枚举值被定义，仅传统形式下的选项是可见的。

Ignore
该选项在使用 Warn 时被忽略，该选项在specs中不可见，并且没有 ‘OPT_’ 对应的值。
SeparateAlias
如果一个选项被标记为 Joined，Separate 和 Alias，该选项将仅作为一个传递分割参数时的时别名，每个参数就是一个普通选项，可以取 ‘OPT_’ 枚举值。因为兼容原因保留的Java -d 选项应该不再被使用。
Warn(message)
如果使用了该选项，输出警告信息 message。message 是一串格式化的字符串，要么是一个 ‘%qs’ 格式的单操作数选项名，要么是传递给 ‘warning’ 函数的多操作数组合。如果一个别名被标记成 Warn 了，那么该别名的目标也应该被标记成 Warn。
Report
选项的说明应该通过-fverbose-asm来打印。
Warning
警告选项，--help 输出中应该有相关内容。该参数目前无效。
Optimization
changed with optimize attributes. 优化选项。--help 输出中应该有相关内容，任何赋给该变量 Var 的值应该在优化等级 optimize 改变时被保存/恢复。
Undocumented
选项有意的不提供文档，并且不应该包括在--help输出中。
Condition(cond)
选项应该只在预处理程序条件cond为真时才被接受。注意即使cond为假时，任何与选项相关的C声明也会存在；cond只是简单的控制选项是否被接受，以及是否在--help输出中被打印。
Save
cl_target_option 结构保存了一个选项的副本，增加 cl_target_option_save 和cl_target_option_restore 函数用来保存和恢复这个选项。
SetByCombined
该选项也可以通过 -ffast-math 一类的组合选项来设置。这导致 gcc_options 结构中保存一个 frontend_set_name 域，其中 name 是当前保持的这个选项（没有前缀 x_）值的名字。这样，前端就可以有办法分辨显式设置，以及不能被组合选项改变值。 例如，有些不直接使用 errno 的前端，通过使用 -ffast-math 和 -fno-fast-math 来改变 -fmath-errno 的值。
