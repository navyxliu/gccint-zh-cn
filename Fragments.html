19 Makefile片段

当使用 configure 脚本配置GCC时，将会从模版文件 Makefile.in 中构建 Makefile 文件。这个时候，会将 config 目录下的makefile片段合在一起。这些片段用来设置不能被autoconf检测计算出的Makefile参数。要合并的片段列表由 config.gcc（以及偶尔由 config.build 和 config.host）设置；参见System Config。

片段命名为 t-target 或者 x-host，取决于它们是否与配置GCC来产生特定目标代码相关，或者配置GCC来运行在特定主机。这里的 target 和 host 是助记符，它们跟系统的规范名称有一些关系，但并不是正式的联系。

如果不存在这些文件，则意味着不需要对给定目标或主机添加什么。大多数目标机需要一些 t-target 片段，不过需要 x-host 片段的很少。

  • Target Fragment: 写 t-target 文件。
  • Host Fragment: 写 x-host 文件。
