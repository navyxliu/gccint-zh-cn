16.7 Predicates

断言用于判断一个 match_operand 或者 match_operator 表达式是否匹 配，以及指令模式是否会被用于那些操作数的组合。GCC有许多机器独立的断言，并且如 果需要的话你可以定义机器特定的断言。根据惯例，与 match_operand 一起使用 的断言名字以 ‘_operand’ 结束，与 match_operator 一起使用的断言名字 以‘_operator’ 结束。

所有断言（从数学意义上）都是具有两个参数的布尔函数：指令模式中的RTL表达式，以 及 match_operand 或 match_operator 指定的机器模式。在这一节里， 第一个参数被称为 op，第二个被称为 mode。断言能够作为普通的两个参数 的函数，由C来调用；这可以用于输出模板或者其它机器特定代码中。

操作数断言能够允许硬件实际上无法接受的操作数，只要约束提供重载的能力，能够修复 它们（see Constraints）。然而，只要断言指定的机器指令要求尽可能的严密，GCC 通常便会生成较好的代码。重载不能修复必须为常量的操作数（立即数）；你必须使用只 允许常量的断言，或者使用额外的条件来加强必要条件。

大多数断言使用统一的方式来处理它们的 mode 参数。如果 mode 为 VOIDmode（未指定的），那么 op 能够具有任意的模式。如果 mode 为其它情况，那么 op 必须具有相同的模式，除非 op 是一个 CONST_INT 或者整数 CONST_DOUBLE。这些RTL表达式总是具有 VOIDmode，所以检测它们的模式匹配是不必要的。相反的，接受 CONST_INT 以及整数 CONST_DOUBLE 的断言，检测存储在常量中的值时， 将会得到适合的模式。

这种行为的断言被称为常规断言。genrecog 能够根据常规断言如何处理模式的 知识来优化指令识别器。它还能够诊断使用常规断言所出现的一些常见错误类型。例如， 常见的使用常规断言而没有指定模式的错误。

对 mode 参数进行不同方式处理的断言被称为特殊断言。通用断言 address_operand 和 pmode_register_operand 是特殊断言。 genrecog 对使用特殊断言的不做任何优化和诊断。
