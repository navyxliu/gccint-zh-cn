4.6 其它运行时库例程

4.6.1 Cache控制函数

— Runtime Function: void __clear_cache (char *beg, char *end)

该函数清除beg和end之间的指令缓存。

4.6.2 分离栈函数和变量

— Runtime Function: void * __splitstack_find (void *segment_arg, void *sp, size_t len, void **next_segment, void **next_sp, void **initial_sp)

当指定参数-fsplit-stack时，调用这个函数可以迭代栈段。可以像这样调用它： 

            void *next_segment = NULL;
            void *next_sp = NULL;
            void *initial_sp = NULL;
            void *stack;
            size_t stack_size;
            while ((stack = __splitstack_find (next_segment, next_sp,
                                               &stack_size, &next_segment,
                                               &next_sp, &initial_sp))
                   != NULL)
              {
          
          
                /* 栈段从stack起始，长度为 stack_size 字节。 */
              }

无法迭代不同线程的栈段。但是有件事是可以做的，那就是一个线程调用此函数时指定参数 segment_arg 和 sp 为 NULL，并且传递参数 next_segment, next_sp, 以及 initial_sp 至一个不同的线程，然后以不论什么方式挂起（suspend）。这时那个不同的线程就可以运行 __splitstack_find 迭代了。只有在第一个线程已经被挂起（suspend）的情况下，第二个线程去调用 __splitstack_find 才能正常工作，否则当第二个线程去查看一个变化中的栈时，什么事都是有可能发生的。

— Variable: __morestack_segments
— Variable: __morestack_current_segment
— Variable: __morestack_initial_sp

参数 -fsplit-stack 的具体实现用到的内部变量。
